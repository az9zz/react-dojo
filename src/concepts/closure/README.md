# Concept: Closure (闭包)

## 什么是闭包？

**闭包 (Closure)** 是指**函数**和其**声明时所在的作用域环境 (Lexical Environment)** 的组合。

通俗地讲，一个函数“记住”了它被创建时的环境。即使这个函数在它被创建的作用域之外被调用，它仍然可以访问和操作该作用域中的变量。

在 JavaScript 中，**当一个函数能够访问其外部函数作用域中的变量时，闭包就产生了**。

## 闭包产生的条件

1.  **函数嵌套**: 存在一个外部函数和一个或多个内部函数。
2.  **内部函数引用外部函数的变量**: 内部函数访问了外部函数作用域中的变量或参数。
3.  **外部函数返回内部函数**: 外部函数将内部函数作为返回值返回，或者以其他方式（如作为参数传递）将其暴露到外部作用域。

## 核心原理：作用域链 (Scope Chain)

当一个函数被创建时，它会创建一个作用域链，这个链指向它自己的作用域、外部函数的作用域，一直到全局作用域。当函数执行并需要访问一个变量时，它会沿着这个作用域链向上查找，直到找到该变量为止。

即使外部函数已经执行完毕并返回，由于其内部函数仍然存在引用（例如被赋值给一个外部变量），垃圾回收机制不会销毁外部函数的变量对象，从而使得内部函数在未来任何时候被调用时，依然能通过作用域链访问到这些变量。

## 闭包的典型应用场景

### 1. 创建私有变量和方法

通过闭包，可以模拟面向对象编程中的私有成员。外部无法直接访问函数内部的变量，只能通过函数返回的接口（方法）来操作。

```javascript
function createCounter() {
  let count = 0 // 私有变量

  return {
    // 返回一个包含公有方法的对象
    increment: function () {
      count++
    },
    getCount: function () {
      return count
    },
  }
}

const counter = createCounter()
counter.increment()
console.log(counter.getCount()) // 1
console.log(counter.count) // undefined (无法直接访问)
```

### 2. 函数柯里化 (Currying)

闭包使得函数可以分步接收参数。

```javascript
function add(x) {
  return function (y) {
    return x + y
  }
}

const add5 = add(5) // add5 是一个闭包，它记住了 x=5
console.log(add5(3)) // 8
```

### 3. 事件处理和回调函数

在循环中为 DOM 元素绑定事件监听器是闭包问题的经典场景（我们之前在 `closure-in-loop` 中已经练习过）。

### 4. 封装功能模块

许多前端库和模块都使用 IIFE (立即执行函数表达式) 包裹代码，形成一个闭包，以避免污染全局作用域。

## 闭包的注意事项

- **内存泄漏**: 如果闭包中引用的外部变量是一个很大的对象，并且这个闭包的生命周期很长，那么这个大对象就无法被垃圾回收，可能导致内存泄漏。在不再需要闭包时，应将其引用设为 `null` (例如，在组件卸载时移除事件监听器)。
- **性能**: 闭包会比非闭包函数多占用一些内存，但在现代 JavaScript 引擎中，这种性能影响通常可以忽略不计。
