# 002. Function Currying (函数柯里化)

## 题目描述

实现一个 `curry` 函数，它接收一个函数作为参数，并返回一个柯里化后的新函数。柯里化后的函数可以被部分调用，每次传入一部分参数，直到接收到所有参数后，才会执行原始函数并返回结果。

## 什么是柯里化？

**柯里化 (Currying)** 是一种将一个接收**多个参数**的函数，转变为一系列接收**单个参数**的函数的技术。

简单来说，就是把 `fn(a, b, c)` 这种形式的调用，转变为 `fn(a)(b)(c)` 的形式。

**示例：**

```javascript
// 原始函数
const add = (a, b, c) => a + b + c

// 柯里化后的函数
const curriedAdd = curry(add)

// 调用方式
curriedAdd(1)(2)(3) // 输出 6
curriedAdd(1, 2)(3) // 输出 6 (更灵活的实现)
curriedAdd(1)(2, 3) // 输出 6 (更灵活的实现)
```

## 柯里化有什么用？

柯里化不仅仅是一种语法上的炫技，它在实际编程中有非常重要的应用，核心价值在于：

### 1. 参数复用 (Partial Application - 部分应用)

这是柯里化最核心的应用。我们可以通过固定一部分参数，来创建一个新的、更具体的函数。

**示例：**

```javascript
// 创建一个通用的请求函数
const request = (method, url, data) => {
  /* ... */
}

const curriedRequest = curry(request)

// 通过固定 method 参数，创建更具体的 get 和 post 函数
const get = curriedRequest('GET')
const post = curriedRequest('POST')

// 使用新函数，代码更简洁、语义更清晰
get('/api/users', { id: 1 })
post('/api/login', { username: 'test' })
```

### 2. 延迟执行 / 惰性求值

柯里化后的函数只有在接收到所有参数后才会执行。这给我们提供了一种“延迟”计算的能力，直到我们真正需要结果时才进行计算。

### 3. 函数组合 (Composition)

在函数式编程中，柯里化是实现函数组合（将多个小函数组合成一个大函数）的重要工具。柯里化的函数更容易进行组合。

## 如何实现 `curry` 函数？

### 核心思想

1. **高阶函数**：`curry` 函数接收一个函数 `fn` 作为参数，并返回一个新函数 `curried`。
2. **闭包**：`curried` 函数需要在一个闭包中“记住”已经接收到的参数。
3. **参数收集**：每次调用 `curried` 函数时，收集传入的参数，并与之前记住的参数合并。
4. **终止条件**：判断当前收集到的参数总数，是否已经达到或超过原始函数 `fn` 所需的参数个数 (`fn.length`)。
   - **如果参数足够**：执行原始函数 `fn` 并返回结果。
   - **如果参数还不够**：**返回一个新的函数**，这个新函数继续等待并收集下一批参数。

### 复杂度分析

- **时间复杂度**: 取决于原始函数 `fn` 的复杂度。`curry` 本身的操作（参数收集、判断）是 O(1) 的。
- **空间复杂度**: O(k)，其中 `k` 是原始函数 `fn` 的参数个数，用于存储已接收的参数。
