# 011. Array Reverse (数组翻转)

## 题目描述

实现一个函数，接收一个数组作为参数，并返回一个元素顺序相反的新数组。同时，也要考虑如何**原地**翻转一个数组。

## 示例

**输入:** `[1, 2, 3, 4, 5]`
**输出:** `[5, 4, 3, 2, 1]`

---

## 解法一：使用原生 `reverse()` 方法

在实际工作中，如果不需要手写实现，这永远是首选。`Array.prototype.reverse()` 是一个**原地**操作，它会修改原始数组。

```javascript
const arr = [1, 2, 3, 4, 5]
arr.reverse()
console.log(arr) // [5, 4, 3, 2, 1]
```

**注意**: 如果不希望修改原数组，需要先创建一个副本：`[...arr].reverse()`。

---

## 解法二：创建一个新数组 (非原地)

这是最容易理解的手写实现，通过遍历原数组并从后往前填充一个新数组。

### 核心思想

1.  创建一个空的结果数组 `result`。
2.  使用 `for` 循环，从原数组的最后一个元素 (`arr.length - 1`) 向前遍历到第一个元素 (`0`)。
3.  在循环中，将当前元素 `arr[i]` `push` 到 `result` 数组的末尾。
4.  返回 `result`。

### 复杂度分析

- **时间复杂度**: O(n)，需要遍历一次数组。
- **空间复杂度**: O(n)，需要一个与原数组等大的新数组。

---

## 解法三：使用 `unshift` (非原地，效率较低)

这种方法比较巧妙，但因为 `unshift` 的性能问题，通常不推荐。

### 核心思想

1.  创建一个空的结果数组 `result`。
2.  **从前向后**遍历原数组。
3.  对于每个元素，使用 `result.unshift(item)` 将其添加到 `result` 数组的**头部**。

### 缺点

- `unshift` 操作的时间复杂度是 O(n)，因为它需要将数组中所有现有元素向后移动一位。这导致整个算法的时间复杂度变成 O(n²)，效率很低。

---

## 解法四：双指针法 (原地翻转 - In-place)

这是面试中最常要求的手写实现，因为它考察了**原地算法**和**双指针**思想，空间效率最高。

### 核心思想

1.  初始化两个指针：`left` 指向数组的第一个元素 (`0`)，`right` 指向数组的最后一个元素 (`arr.length - 1`)。
2.  使用 `while` 循环，条件是 `left < right`。当 `left` 和 `right` 相遇或交错时，翻转完成。
3.  在循环中：
    - 交换 `arr[left]` 和 `arr[right]` 的值。
    - 将 `left` 指针向右移动 (`left++`)。
    - 将 `right` 指针向左移动 (`right--`)。
4.  由于是原地操作，函数可以没有返回值 (或者返回修改后的原数组)。

### 复杂度分析

- **时间复杂度**: O(n)，我们只需要遍历数组的一半。
- **空间复杂度**: O(1)，只需要常数级别的额外空间来存储指针和临时变量。
