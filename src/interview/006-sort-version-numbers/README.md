# 005. Sort Version Numbers (版本号排序)

## 题目描述

给定一个包含版本号的字符串数组 `versions`，请你对它进行排序。

版本号的格式为 `major.minor.patch` (主版本号.次版本号.修订号)，每一段都是非负整数。

排序规则：

- 从左到右，逐段比较版本号的数字大小。
- 如果某一段数字不同，则数字大的版本号更大。
- 如果所有已比较的段都相同，则段数多的版本号更大（例如 `1.2.1` > `1.2`）。

## 示例

**示例 1：**

- **输入:** `['1.10.2', '1.2.0', '1.1.3', '0.1.0', '1.2.1']`
- **输出:** `['0.1.0', '1.1.3', '1.2.0', '1.2.1', '1.10.2']`

**示例 2：**

- **输入:** `['1.0.1', '1', '1.0']`
- **输出:** `['1', '1.0', '1.0.1']`
- **解释:** `1` 相当于 `1.0.0`, `1.0` 相当于 `1.0.0`。但 `1.0.1` 段数更多，所以更大。这里假设 `1` 和 `1.0` 在排序中可视为相等，但通常规范认为段数多的更大。

## 设计思路

这个问题的核心是自定义 `Array.prototype.sort` 的比较函数 `compareFunction(a, b)`。

`sort` 方法会根据 `compareFunction` 的返回值来决定元素的顺序：

- 如果返回 `> 0`，则 `b` 排在 `a` 前面。
- 如果返回 `< 0`，则 `a` 排在 `b` 前面。
- 如果返回 `0`，则 `a` 和 `b` 的相对位置不变。

我们的任务就是实现这个比较函数。

### 算法步骤

1.  **预处理**:
    - 接收两个版本号字符串 `a` 和 `b`。
    - 使用 `split('.')` 将它们转换为数字数组。例如，`'1.10.2'` -> `['1', '10', '2']`。
    - 为了方便比较，最好将字符串数组转换为数字数组，例如 `['1', '10', '2']` -> `[1, 10, 2]`。

2.  **逐段比较**:
    - 确定需要比较的轮次，即两个数字数组中较长的那个的长度 `maxLength = Math.max(arrA.length, arrB.length)`。
    - 使用 `for` 循环，从 `i = 0` 到 `maxLength - 1` 进行遍历。
    - 在循环中，获取 `arrA[i]` 和 `arrB[i]` 的值。如果某个数组已经遍历完（`i` 超出其长度），则其对应的值视为 `0`。这可以优雅地处理 `1.2` 和 `1.2.1` 这种不等长的情况。
    - 比较两个值 `numA` 和 `numB`。
      - 如果 `numA > numB`，说明版本 `a` 更大，比较函数返回 `1`。
      - 如果 `numA < numB`，说明版本 `b` 更大，比较函数返回 `-1`。
      - 如果相等，则继续下一轮循环。

3.  **最终返回**:
    - 如果循环结束后所有段都相等，说明两个版本号等价，返回 `0`。

### 封装与调用

1.  将上述比较逻辑封装成一个独立的函数 `compareVersions(a, b)`。
2.  在主排序函数中，调用 `versions.sort(compareVersions)`。
    - **注意**: `sort` 方法会**原地修改**数组。为了不改变原始输入，最好先创建一个副本 `[...versions].sort(...)`。

## 复杂度分析

- **时间复杂度**: O(N _ L _ logN)，其中 N 是版本号的数量，L 是版本号的最大段数。`logN` 是 `sort` 方法的平均复杂度，每次比较的时间复杂度是 L。
- **空间复杂度**: O(N _ L) 或 O(L)，取决于 `sort` 方法的内部实现。如果需要创建副本，则为 O(N _ L)。
