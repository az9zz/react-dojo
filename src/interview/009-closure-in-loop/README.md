# 009. The Closure in a Loop Problem (循环中的闭包问题)

## 问题描述

看下面这段代码，它的预期输出是什么？实际输出又是什么？如何修复它以达到预期输出？

**有问题的代码:**

```javascript
for (var i = 0; i < 5; i++) {
  setTimeout(function () {
    console.log(i)
  }, i * 1000)
}
```

- **预期输出**: 每隔一秒，依次输出 `0`, `1`, `2`, `3`, `4`。
- **实际输出**: 每隔一秒，依次输出 `5`, `5`, `5`, `5`, `5`。

## 问题成因分析

这个问题的根源在于 `var` 的**函数作用域**和 `setTimeout` 的**异步执行**机制的结合。

1.  **`var` 的作用域**: 使用 `var` 声明的变量 `i` 存在于**全局作用域**或**函数作用域**中。在 `for` 循环中，`i` 只有一个实例，它不属于循环体的块级作用域。
2.  **异步执行**: `setTimeout` 会将回调函数放入事件队列中，等待主线程的同步代码（即 `for` 循环）执行完毕后才会执行。
3.  **闭包陷阱**: `setTimeout` 的回调函数形成了一个闭包，它“记住”了其创建时所在作用域的变量 `i`。但由于 `for` 循环中只有一个 `i` 实例，所有五个回调函数都共享并引用了**同一个** `i` 变量。
4.  **执行流程**:
    - `for` 循环**瞬间**执行完毕。此时，`i` 的值已经变成了 `5`。
    - 之后，`setTimeout` 的回调函数才开始被逐个从事件队列中取出执行。
    - 当它们执行 `console.log(i)` 时，它们访问的是那个已经变成 `5` 的全局（或函数作用域）变量 `i`。因此，所有回调函数都输出了 `5`。

---

## 解决方案

### 解法一：使用 `let` (ES6 最佳实践)

这是最简单、最现代的解决方案。`let` 具有**块级作用域**。

```javascript
for (let i = 0; i < 5; i++) {
  setTimeout(function () {
    console.log(i)
  }, i * 1000)
}
```

**原理**:
在每次 `for` 循环的迭代中，`let` 都会创建一个**新的**、只在当前循环块内有效的变量 `i`。`setTimeout` 的回调函数在每次迭代中都捕获了**一个不同**的 `i` 实例，每个实例都保存了当时 `i` 的值 (`0`, `1`, `2`, `3`, `4`)。

### 解法二：使用立即执行函数表达式 (IIFE) (ES5 经典解法)

在没有 `let` 的 ES5 时代，这是最标准的解决方案。

```javascript
for (var i = 0; i < 5; i++) {
  ;(function (j) {
    setTimeout(function () {
      console.log(j)
    }, j * 1000)
  })(i)
}
```

**原理**:
我们使用一个立即执行函数 (IIFE) 来创建一个**新的函数作用域**。

- 在每次循环中，我们将当前的 `i` 的**值**作为参数传递给 IIFE。
- IIFE 内部的参数 `j` 接收了这个值，`j` 是一个新的、属于该次 IIFE 执行的私有变量。
- `setTimeout` 的回调函数现在捕获的是这个私有的、在每次循环中都不同的 `j`。

### 解法三：利用 `setTimeout` 的第三个参数

这是一个比较巧妙但不那么通用的方法。

```javascript
for (var i = 0; i < 5; i++) {
  setTimeout(
    function (j) {
      console.log(j)
    },
    i * 1000,
    i,
  )
}
```

**原理**:
`setTimeout` 的第三个及以后的参数，会作为参数传递给其回调函数。在每次循环中，我们将当前的 `i` 的**值**作为第三个参数传给 `setTimeout`。这个值会被 `setTimeout` 内部机制保存下来，并在执行回调时传递给回调函数的第一个参数 `j`。

### 解法四：使用函数柯里化或绑定

可以创建一个返回新函数的函数，本质上和 IIFE 类似，都是创建新的作用域。

```javascript
function createLogger(i) {
  return function () {
    console.log(i)
  }
}

for (var i = 0; i < 5; i++) {
  setTimeout(createLogger(i), i * 1000)
}
```

**原理**:
每次调用 `createLogger(i)` 都会创建一个新的执行上下文，参数 `i` 被保存在这个上下文中。返回的函数形成了一个闭包，捕获了这个被保存的 `i`。
