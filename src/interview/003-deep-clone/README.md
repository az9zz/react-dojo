# 002. Deep Clone (深拷贝)

## 题目描述

实现一个函数，可以对一个 JavaScript 对象进行深拷贝。

**深拷贝 vs. 浅拷贝:**

- **浅拷贝 (Shallow Clone)**: 只复制对象的第一层属性。如果属性值是引用类型（如对象、数组），那么拷贝的只是这个引用（内存地址），而不是引用指向的实际内容。修改拷贝后的对象会影响原对象。常见的浅拷贝方法有 `Object.assign({}, obj)` 和 `{...obj}`。
- **深拷贝 (Deep Clone)**: 递归地复制一个对象及其所有子对象。拷贝后的对象是一个全新的、独立的实体，与原对象没有任何引用关系。修改拷贝后的对象**不会**影响原对象。

## 示例

```javascript
const original = {
  a: 1,
  b: 'hello',
  c: [1, 2, 3],
  d: {
    e: new Date(),
    f: function () {
      console.log('f')
    },
    g: /test/gi,
  },
  h: null,
  i: undefined,
}

// 添加循环引用，这是一个棘手的情况
original.self = original

const cloned = deepClone(original)

console.log(cloned.d.e === original.d.e) // false (Date 对象被新创建)
console.log(cloned.c === original.c) // false (数组被新创建)
console.log(cloned.self === cloned) // true (循环引用被正确处理)

original.c[0] = 99
console.log(original.c) // [99, 2, 3]
console.log(cloned.c) // [1, 2, 3] (修改原对象不影响克隆对象)
```

---

## 解法一：`JSON.parse(JSON.stringify(obj))` (简陋但常用)

这是最简单、最快捷的深拷贝方式，但限制非常多。

### 优点

- 代码极其简单，一行搞定。
- 能处理常见的 JSON 安全数据类型（数字、字符串、布尔、数组、普通对象）。

### 致命缺点

- **会忽略 `undefined`、`function`、`Symbol`** 类型的属性。
- **无法处理循环引用**，会直接报错 `TypeError: Converting circular structure to JSON`。
- **会把 `Date` 对象转换为字符串**，丢失原始类型。
- **会把 `RegExp` 正则表达式对象转换为空对象 `{}`**。
- **无法拷贝对象的原型链**。

**结论**: 只能用于拷贝纯粹的、无函数的、无特殊对象的 JSON 数据。面试中可以提及，但必须清晰地指出其弊端。

---

## 解法二：递归实现 (基础版)

这是面试中最常要求手写的版本，考察递归基础。这个版本先不考虑循环引用和其他复杂类型。

### 核心思想

1. 判断输入值的类型，如果不是对象或为 `null`，直接返回该值（这是递归的终止条件）。
2. 根据输入值的类型（数组或对象），创建一个新的空数组或空对象作为拷贝结果。
3. 遍历原对象的所有 `key` (使用 `for...in` 并配合 `hasOwnProperty` 检查)。
4. 对每个 `key` 对应的 `value`，**递归调用**深拷贝函数，并将返回的结果赋值给新对象的相应 `key`。
5. 返回拷贝结果。

### 缺点

- 无法处理循环引用，会导致无限递归，最终栈溢出。
- 未处理 `Date`, `RegExp` 等特殊对象类型。

---

## 解法三：递归实现 (进阶版 - 解决循环引用)

在基础版上增加一个机制来检测和处理循环引用。

### 核心思想

我们需要一个地方来存储已经拷贝过的对象及其对应的拷贝结果。`Map` 或 `WeakMap` 是理想的选择。

- 使用 `WeakMap` 更优，因为它对键是弱引用，当原对象被垃圾回收时，`WeakMap` 中的对应条目也会被自动移除，有助于防止内存泄漏。

### 算法步骤

1. 创建一个 `WeakMap` 作为缓存 `cache`。
2. 在深拷贝函数内部，首先检查 `cache` 中是否已经存在当前要拷贝的对象 `obj`。
   - **如果存在**，说明遇到了循环引用，直接从 `cache` 中返回对应的拷贝结果，中断递归。
3. 如果 `cache` 中不存在，继续执行基础版的逻辑。
4. 在创建了新的拷贝对象 `newObj` 之后，**立刻**将 `[obj, newObj]` 这对键值存入 `cache` 中。
5. 继续遍历和递归拷贝子属性。
6. 返回 `newObj`。

---

## 解法四：递归实现 (完整版 - 处理多种数据类型)

在进阶版的基础上，进一步处理 `Date`、`RegExp`、`Map`、`Set` 等更多数据类型。

### 核心思想

在递归拷贝之前，增加对各种常见引用类型的判断，并为它们提供专门的拷贝逻辑。

### 算法步骤 (在进阶版基础上扩展)

1. 在递归函数开头，增加判断：
   - 如果是 `Date` 对象，返回 `new Date(obj)`。
   - 如果是 `RegExp` 对象，返回 `new RegExp(obj.source, obj.flags)`。
   - 如果是 `Map` 对象，创建一个新 `Map`，并递归拷贝其键值对。
   - 如果是 `Set` 对象，创建一个新 `Set`，并递归拷贝其值。
2. 对于普通的数组和对象，沿用之前的逻辑。

这个版本已经非常接近 `lodash` 等库中 `cloneDeep` 的实现了（虽然 lodash 的实现要复杂得多，考虑了更多边界情况）。
