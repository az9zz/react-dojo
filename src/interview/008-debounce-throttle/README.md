# 006. Debounce & Throttle (防抖与节流)

## 问题描述

在前端开发中，某些事件（如 `resize`, `scroll`, `mousemove`, `input`) 会在短时间内被高频触发。如果事件的回调函数涉及复杂计算或 DOM 操作，频繁调用会导致性能问题。为了解决这个问题，我们需要对函数的触发频率进行限制。**防抖 (Debounce)** 和 **节流 (Throttle)** 就是两种最常用的高阶函数技术。

---

## 防抖 (Debounce)

### 核心思想

**在事件被触发 n 秒后再执行回调，如果在这 n 秒内又被触发，则重新计时。**

就像电梯关门：每次有人进来，电梯的关门倒计时都会重置。只有当最后一个人进来后，等待一段时间，门才会关上。

### 应用场景

- **搜索框输入提示 (Autocomplete)**: 用户在输入时，我们不希望每输入一个字符就发一次请求，而是在用户停止输入一小段时间后再发。
- **窗口大小调整 (window.resize)**: 不希望在拖拽窗口的过程中不断触发重新布局，而是在停止拖拽后触发一次。
- **表单验证**: 在用户填写完一项内容后再进行验证，而不是每输入一个字符就验证。

### 实现要点

1.  **返回一个新函数**: 防抖函数本身是一个高阶函数，它接收一个目标函数 `fn` 和一个延迟时间 `delay`，并返回一个新的、经过防抖处理的函数。
2.  **定时器 (Timer)**: 使用 `setTimeout` 来实现延迟执行。我们需要一个变量 (例如 `timer`) 来保存 `setTimeout` 返回的 ID。
3.  **清除旧的定时器**: 在新函数被调用时，首先要用 `clearTimeout(timer)` 清除上一次设置的定时器。这正是“重新计时”的核心。
4.  **设置新的定时器**: 清除后，再设置一个新的 `setTimeout`，在 `delay` 毫秒后执行原始函数 `fn`。
5.  **`this` 和 `arguments`**: 为了让防抖函数能正确处理原始函数的 `this` 指向和参数，需要使用 `function` 关键字定义返回的函数（或使用箭头函数并保存外部的 `this`），并使用 `...args` 或 `arguments` 来传递参数。

---

## 节流 (Throttle)

### 核心思想

**在 n 秒内，事件回调函数最多只能执行一次。如果在 n 秒内被多次触发，只有第一次（或最后一次）有效。**

就像技能冷却：一个技能使用后，需要等待一段冷却时间 (CD) 才能再次使用。在冷却期间，无论你按多少次技能键，都无法施放。

### 应用场景

- **页面滚动事件 (scroll)**: 监听滚动位置，实现懒加载、返回顶部按钮的显隐等。我们不需要实时响应，每隔 200ms 检查一次就足够了。
- **拖拽 (drag)**: 在拖拽过程中，每隔一段时间更新元素的位置。
- **高频点击**: 防止用户快速点击按钮，重复提交表单。

### 实现要点

#### 版本一：定时器实现 (触发后延迟执行)

1.  **返回一个新函数**。
2.  **标志位 (Flag)**: 使用一个标志位 (例如 `timer` 或 `inThrottle`) 来判断当前是否处于“冷却”状态。
3.  **执行逻辑**: 当新函数被调用时，检查标志位。
    - 如果处于冷却状态 (标志位为 `true`)，则直接 `return`，什么都不做。
    - 如果不在冷却状态 (标志位为 `false`)，则设置一个 `setTimeout`，在 `delay` 毫秒后执行原始函数 `fn`，并将标志位设回 `false`。同时将标志位设为 `true`，进入冷却。

#### 版本二：时间戳实现 (立即执行)

1.  **返回一个新函数**。
2.  **记录上次执行时间**: 使用一个变量 (例如 `lastTime`) 记录上一次执行函数的时间戳。
3.  **执行逻辑**: 当新函数被调用时，获取当前时间戳 `now`。
    - 比较 `now - lastTime` 是否大于等于 `delay`。
    - 如果大于，说明冷却时间已过，可以执行。立即执行原始函数 `fn`，并更新 `lastTime = now`。
    - 如果小于，则什么都不做。

## 总结

- **防抖**: **“王者归来”**，等你操作完，过一会再响应。关注的是**结果**。
- **节流**: **“技能冷却”**，按固定频率响应。关注的是**过程**。
