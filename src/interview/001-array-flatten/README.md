# 001. Array Flatten (扁平化嵌套数组)

## 题目描述

实现一个函数，将一个嵌套多层的数组 `array` "拉平"（flatten），变成一个一维数组。

## 示例

**示例 1：**

- **输入:** `[1, [2, [3, 4], 5], 6]`
- **输出:** `[1, 2, 3, 4, 5, 6]`

**示例 2：**

- **输入:** `[1, 2, [3, 4, [5]]]`
- **输出:** `[1, 2, 3, 4, 5]`

**示例 3：**

- **输入:** `[1, [2, 3], [4, [5, 6]]]`
- **输出:** `[1, 2, 3, 4, 5, 6]`

---

## 解法一：ES6 `flat()` 方法 (终极解法)

在实际工作中，如果环境允许（现代浏览器和 Node.js 都支持），这永远是最佳选择。简洁、高效、可读性强。

### 核心思想

`Array.prototype.flat(depth)` 方法会按照一个可指定的深度 `depth` 递归地将所有子数组合并到新数组中。

- `depth`: 指定要提取嵌套数组的结构深度，默认为 `1`。
- 我们可以传入 `Infinity` 作为 `depth`，这样无论数组嵌套多少层，它都可以完全扁平化。

### 复杂度分析

- **时间复杂度**: O(n)，其中 `n` 是数组中所有元素的总数。
- **空间复杂度**: O(n)，需要返回一个新数组。

---

## 解法二：递归 (经典思路)

这是最经典的面试思路，考察对递归的理解。

### 核心思想

遍历数组，对每个元素进行判断：

- 如果元素不是数组，直接将其加入结果数组。
- 如果元素是数组，**递归调用**扁平化函数处理这个子数组，然后将返回的结果合并到最终的结果数组中。

### 算法步骤

1. 初始化一个空的结果数组 `result`。
2. 遍历输入数组 `arr`。
3. 对于每个元素 `item`，使用 `Array.isArray(item)` 判断其类型。
4. 如果是数组，使用扩展运算符 `...` 或 `concat` 将递归调用的结果合并到 `result`。
5. 如果不是数组，直接 `push` 到 `result`。
6. 返回 `result`。

---

## 解法三：使用 `reduce` (函数式编程)

这种方法是递归思想的函数式编程体现，代码更简洁，但理解起来可能需要一点思考。

### 核心思想

使用 `Array.prototype.reduce()` 方法来遍历数组并构建最终的扁平化数组。

- `reduce` 的回调函数接收两个参数：`accumulator` (累加器，即我们正在构建的结果数组 `result`) 和 `currentValue` (当前元素 `item`)。
- 在回调函数中，我们同样判断 `item` 是否是数组。
  - 如果是，递归调用扁平化函数，并将其结果与 `accumulator` 合并。
  - 如果不是，直接将 `item` 与 `accumulator` 合并。
- `reduce` 的初始值是一个空数组 `[]`。

---

## 解法四：迭代 + 栈 (非递归思路)

这种方法使用栈来模拟递归的调用过程，可以避免在嵌套层数非常深时可能出现的“栈溢出”问题。

### 核心思想

为了在不使用递归的情况下处理嵌套结构，并保持元素的原始顺序，我们需要巧妙地利用栈的 **LIFO (后进先出)** 特性。

1. 创建一个栈 `stack`，并将**原数组的副本** `[...arr]` 直接作为初始内容。（注意，这里不是把整个数组作为一个元素压入，而是将其所有元素展开压入）。
2. 创建一个结果数组 `result`。
3. 当栈不为空时，循环执行以下操作：
   - 从栈顶弹出一个元素 `item`。
   - 判断 `item` 的类型：
     - 如果**是数组**，则将其所有元素**正序**地 `push` 回栈顶。
     - 如果**不是**数组，则使用**头插法 (`unshift`)** 将其添加到 `result` 数组的开头。

### 为什么这个组合是正确的？

我们来模拟一下 `[1, [2, 3], 4]` 的过程：

1. **初始栈**: `[1, [2, 3], 4]`
2. **`pop` -> `4`**: 非数组。`result.unshift(4)` -> `result` is `[4]`
3. **`pop` -> `[2, 3]`**: 是数组。**正序**将其元素 `2`, `3` 压回栈。
4. **当前栈**: `[1, 2, 3]`
5. **`pop` -> `3`**: 非数组。`result.unshift(3)` -> `result` is `[3, 4]`
6. **`pop` -> `2`**: 非数组。`result.unshift(2)` -> `result` is `[2, 3, 4]`
7. **`pop` -> `1`**: 非数组。`result.unshift(1)` -> `result` is `[1, 2, 3, 4]`

通过 **“后进”** (子数组的元素) 和 **“头插”** 的组合，我们实现了对深层元素的优先处理，并最终恢复了正确的顺序。

**备选逻辑**: 另一种完全正确的思路是 **“倒序入栈 + 尾插法(`push`)”**，这种方法在性能上可能略优，但代码实现上需要对子数组进行一次反转操作。

## 解法五：`toString()` + `split()` (巧妙但不推荐的 "Hack")

这是一种非常取巧的方法，利用了数组的 `toString()` 特性。

### 核心思想

1. 调用数组的 `toString()` 方法，它会自动将多维数组转换为一个用逗号分隔的字符串。例如 `[1, [2, 3]]` 会变成 `"1,2,3"`。
2. 使用 `split(',')` 将这个字符串重新变回一个字符串数组 `['1', '2', '3']`。
3. 遍历这个字符串数组，将每个元素用 `Number()` 或 `parseInt()` 转换回数字。

### 缺点

- **只能处理纯数字数组**。如果数组中包含 `null`, `undefined`, 对象或其他类型，`toString()` 的结果会不符合预期。
- 性能通常不如其他方法。
- 在面试中可以作为一种“脑筋急转弯”提及，但**绝不能作为主要或推荐解法**。

## 带深度的扁平化 (Version 2.0)

现在，我们给问题增加一个难度：实现一个 `flatten(array, depth)` 函数，其中 `depth` 参数指定了需要扁平化的深度。`depth` 默认为 1。

**示例:**

- `flatten([1, [2, [3, 4]]], 1)` -> `[1, 2, [3, 4]]`
- `flatten([1, [2, [3, 4]]], 2)` -> `[1, 2, 3, 4]`

---

### 解法一 (带深度)：ES6 `flat()` (原生支持)

`flat()` 方法本身就接收 `depth` 参数，所以这依然是最佳解法。

```javascript
arr.flat(depth)
```

---

### 解法二 (带深度)：递归

#### 改造思路

我们需要在递归函数中传递 `depth` 参数。

- 每当遇到一个数组并决定要“进入”递归时，我们就将 `depth` 减 1。
- 如果 `depth` 大于 0，我们就继续递归扁平化子数组。
- 如果 `depth` 等于 0，我们就停止深入，直接将当前子数组原样保留。

```javascript
function flattenByRecursionDepth(arr, depth = 1) {
  // ...
  if (Array.isArray(item) && depth > 0) {
    // 深度足够，继续递归，并将深度减一
    result.push(...flattenByRecursionDepth(item, depth - 1))
  } else {
    // 深度不够或 item 不是数组，直接 push
    result.push(item)
  }
  // ...
}
```

---

### 解法三 (带深度)：使用 `reduce`

#### 改造思路

与递归版本类似，在 `reduce` 的回调函数中检查 `depth`。

```javascript
arr.reduce((acc, cur) => {
  if (Array.isArray(cur) && depth > 0) {
    // ... 递归调用，并传递 depth - 1
    return acc.concat(flattenByReduceDepth(cur, depth - 1))
  }
  return acc.concat(cur)
}, [])
```

---

### 解法四 (带深度)：迭代 + 栈

#### 改造思路

这是改造起来最复杂，但也最能体现对迭代理解的。既然栈是用来模拟递归调用栈的，那么除了要将数组元素压入栈，我们还必须**将当前元素的深度也一同压入栈中**。

1. 修改栈的数据结构：栈中存放的不再是`item`，而是一个元组 `[item, currentDepth]`。
2. 初始时，将原数组的每个元素和初始深度 `1` 组合后入栈。
3. 从栈顶弹出一个 `[item, d]`。
4. 判断 `item` 类型：
   - 如果不是数组，直接加入结果。
   - 如果是数组：
     - **检查深度**：如果当前深度 `d` **小于** 目标 `depth`，则说明这个子数组还需要继续扁平化。将其内部元素与 `d + 1` 的深度组合后，再压入栈。
     - 如果当前深度 `d` **已经达到或超过** `depth`，则停止扁平化，直接将这个 `item` 数组作为一个整体加入结果。
