# 206. Reverse Linked List (反转链表)

## 题目描述

给你单链表的头节点 `head` ，请你反转链表，并返回反转后的链表。

## 示例

**示例 1:**

- **输入:** `head = [1,2,3,4,5]`
- **输出:** `[5,4,3,2,1]`

**示例 2:**

- **输入:** `head = [1,2]`
- **输出:** `[2,1]`

**示例 3:**

- **输入:** `head = []`
- **输出:** `[]`

## 约束条件

- 链表中节点的数目范围是 `[0, 5000]`
- `-5000 <= Node.val <= 5000`

---

## 解题思路

反转链表的核心在于改变每个节点的 `next` 指针，使其指向前一个节点。直接操作会导致后续节点“丢失”，因此我们需要借助额外的指针来保存状态。最经典的解法是**迭代法（三指针法）**。

1.  **需要三个指针**:
    - `prev` (previous): 指向当前节点的前一个节点。初始时，头节点的前一个节点是 `null`，所以 `prev` 初始化为 `null`。
    - `curr` (current): 指向当前正在处理的节点。初始时，我们从头节点 `head` 开始处理。
    - `nextTemp` (next temporary): 临时存储当前节点的下一个节点。这是**至关重要**的一步，因为我们马上就要修改 `curr.next` 的指向，如果不提前保存，就会丢失后面的整个链表。

2.  **迭代过程**:
    - 当 `curr` 不为 `null` 时，循环继续。
    - 在循环内部，按以下四步操作：
      1.  **保存 next**: `nextTemp = curr.next`。
      2.  **反转指针**: `curr.next = prev`。这是反转的核心，将当前节点的 `next` 指向前一个节点。
      3.  **`prev` 指针后移**: `prev = curr`。为下一次循环做准备。
      4.  **`curr` 指针后移**: `curr = nextTemp`。处理下一个节点。

3.  **结束条件**:
    - 当循环结束时，`curr` 变为 `null`，说明已经处理完所有节点。此时，`prev` 指针正好指向原链表的最后一个节点，也就是反转后新链表的头节点。我们返回 `prev` 即可。

4.  **复杂度分析**:
    - **时间复杂度**: O(n)，其中 `n` 是链表的长度。我们只需要遍历链表一次。
    - **空间复杂度**: O(1)。我们只使用了常数个额外的指针变量，与链表长度无关。
