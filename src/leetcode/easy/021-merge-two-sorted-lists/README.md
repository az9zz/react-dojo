# 21. Merge Two Sorted Lists & 88. Merge Sorted Array

这两个问题都是“合并有序序列”的经典变体，一个处理链表，一个处理数组。它们都使用了“双指针”的思想，但具体实现因数据结构的特性而异。

---

## 21. Merge Two Sorted Lists (合并两个有序链表)

### 题目描述

将两个升序链表合并为一个新的 **升序** 链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的。

### 示例

- **输入:** `list1 = [1,2,4]`, `list2 = [1,3,4]`
- **输出:** `[1,1,2,3,4,4]`

### 解题思路：迭代 + 哨兵节点

1.  **哨兵节点 (Dummy Node)**: 创建一个哨兵节点 `dummyHead`。它的作用是作为新链表的“伪头节点”，可以极大地简化边界情况的处理（例如，新链表为空时如何插入第一个节点）。我们最终返回的是 `dummyHead.next`。
2.  **当前指针 (Current Pointer)**: 创建一个 `currentNode` 指针，初始指向 `dummyHead`。`currentNode` 用于在新链表上移动和添加新节点。
3.  **双指针遍历**:
    - 使用两个指针 `p1` 和 `p2`，分别指向 `list1` 和 `list2` 的头节点。
    - 当 `p1` 和 `p2` 都不为空时，比较 `p1.val` 和 `p2.val` 的大小。
    - 将值较小的那个节点链接到 `currentNode.next`，然后将该节点的指针 (`p1` 或 `p2`) 和 `currentNode` 指针都向后移动一步。
4.  **处理剩余部分**:
    - 当循环结束后，`p1` 和 `p2` 中最多只有一个还不为空。
    - 将不为空的那个链表的剩余部分直接链接到 `currentNode.next`。
5.  **返回结果**: 返回 `dummyHead.next`。

### 复杂度分析

- **时间复杂度**: O(m + n)，其中 m 和 n 分别是两个链表的长度。
- **空间复杂度**: O(1)，我们只使用了常数级别的额外空间（几个指针）。

## 88. Merge Sorted Array (合并两个有序数组)

### 题目描述

给你两个按 **非递减顺序** 排列的整数数组 `nums1` 和 `nums2`，另有两个整数 `m` 和 `n` ，分别表示 `nums1` 和 `nums2` 中的元素数目。

请你 **原地** 合并 `nums2` 到 `nums1` 中，使合并后的数组同样按 **非递减顺序** 排列。

**注意**：最终，合并后的数组不应由函数返回，而是存储在数组 `nums1` 中。为了应对这种情况，`nums1` 的初始长度为 `m + n`，其中前 `m` 个元素表示应合并的元素，后 `n` 个元素为 `0` ，应忽略。

### 示例

- **输入:** `nums1 = [1,2,3,0,0,0]`, `m = 3`, `nums2 = [2,5,6]`, `n = 3`
- **输出:** `[1,2,2,3,5,6]`

### 解题思路：从后往前双指针 (避免覆盖)

由于要求是**原地**修改 `nums1`，如果我们从前往后合并，那么在 `nums1` 中插入元素时，会覆盖掉 `nums1` 中尚未被比较的原始元素。

正确的做法是**从后往前**填充 `nums1`。

1.  **三个指针**:
    - `p1`: 指向 `nums1` 中有效元素的末尾，初始为 `m - 1`。
    - `p2`: 指向 `nums2` 的末尾，初始为 `n - 1`。
    - `p`: 指向 `nums1` 数组的最终末尾，初始为 `m + n - 1`。这个指针是用来放置最终结果的。
2.  **从后往前遍历**:
    - 当 `p1` 和 `p2` 都有效时 (`>= 0`)，比较 `nums1[p1]` 和 `nums2[p2]` 的大小。
    - 将**较大**的那个元素复制到 `nums1[p]` 的位置。
    - 将被复制的那个元素的指针和 `p` 指针都向前移动一步。
3.  **处理剩余部分**:
    - 当循环结束后，如果 `p2` 指针还有效 (`>= 0`)，说明 `nums1` 的有效元素已经全部放好，但 `nums2` 中还有剩余的（并且是最小的）元素。
    - 将 `nums2` 中剩余的元素依次复制到 `nums1` 的头部。
    - （`p1` 指针不需要处理，因为它指向的元素已经在 `nums1` 中了）。
4.  **返回结果**: 无需返回，`nums1` 已被原地修改。

### 复杂度分析

- **时间复杂度**: O(m + n)。
- **空间复杂度**: O(1)。
