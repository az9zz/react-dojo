# 141. Linked List Cycle (环形链表)

## 题目描述

给你一个链表的头节点 `head` ，判断链表中是否有环。

如果链表中有某个节点，可以通过连续跟踪 `next` 指针再次到达，则链表中存在环。 为了表示给定链表中的环，评测系统内部使用一个整数 `pos` 来表示链表尾连接到链表中的位置（索引从 0 开始）。**注意：`pos` 不作为参数进行传递**。仅仅是为了标识链表的实际情况。

如果链表中存在环，则返回 `true` 。否则，返回 `false` 。

## 示例

**示例 1：**
![示例1](https://assets.leetcode.com/uploads/2018/12/07/circularlinkedlist.png)

- **输入:** `head = [3,2,0,-4]`, `pos = 1`
- **输出:** `true`
- **解释:** 链表中有一个环，其尾部连接到第二个节点。

**示例 2：**
![示例2](https://assets.leetcode-cn.com/aliyun-lc-upload/uploads/2018/12/07/circularlinkedlist_test2.png)

- **输入:** `head = [1,2]`, `pos = 0`
- **输出:** `true`
- **解释:** 链表中有一个环，其尾部连接到第一个节点。

**示例 3：**
![示例3](https://assets.leetcode.com/uploads/2018/12/07/circularlinkedlist_test3.png)

- **输入:** `head = [1]`, `pos = -1`
- **输出:** `false`
- **解释:** 链表中没有环。

## 约束条件

- 链表中节点的数目范围是 `[0, 10^4]`
- `-10^5 <= Node.val <= 10^5`
- `pos` 为 `-1` 或者链表中的一个 **有效索引** 。

---

## 解题思路

判断链表是否有环，最经典、最高效的方法是 **Floyd's Tortoise and Hare Algorithm**，即 **快慢指针法**。

1.  **核心思想**：
    想象一个环形跑道，一个跑得快的人（快指针）和一个跑得慢的人（慢指针）同时从起点出发。只要跑道是环形的，跑得快的人最终一定会从后面“追上”并“套圈”跑得慢的人。如果跑道不是环形的，快的人会先到达终点。

2.  **算法步骤**：
    - 创建两个指针：`slow` 和 `fast`，都初始化指向链表的头节点 `head`。
    - 进入一个循环，循环条件是 `fast` 指针和 `fast.next` 不能为空（保证快指针可以一次走两步）。
    - 在循环内部：
      - `slow` 指针每次向前移动一步：`slow = slow.next`。
      - `fast` 指针每次向前移动两步：`fast = fast.next.next`。
    - 在移动后，立刻判断 `slow` 和 `fast` 是否指向同一个节点 (`slow === fast`)。
      - 如果它们相遇了，说明链表中存在一个环，立即返回 `true`。
    - 如果循环正常结束（即 `fast` 或 `fast.next` 变成了 `null`），说明快指针到达了链表的末尾，链表中没有环，返回 `false`。

3.  **复杂度分析**:
    - **时间复杂度**: O(n)，其中 `n` 是链表的节点数。
    - **空间复杂度**: O(1)，因为我们只使用了两个额外的指针。
