# 146. LRU Cache (LRU 缓存机制)

## 题目描述

请你设计并实现一个满足 [LRU (最近最少使用) 缓存](<https://en.wikipedia.org/wiki/Cache_replacement_policies#Least_recently_used_(LRU)>) 约束的数据结构。

实现 `LRUCache` 类：

- `LRUCache(int capacity)`: 以 **正整数** 作为容量 `capacity` 初始化 LRU 缓存。
- `int get(int key)`: 如果关键字 `key` 存在于缓存中，则返回关键字的值，否则返回 `-1`。
- `void put(int key, int value)`: 如果关键字 `key` 已经存在，则变更其数据值 `value`；如果不存在，则向缓存中插入该组 `key-value`。如果插入操作导致关键字数量超过 `capacity`，则应该 **逐出** 最近最少使用的关键字。

函数 `get` 和 `put` 必须以 **O(1)** 的平均时间复杂度运行。

## 示例

**输入:**

```
["LRUCache", "put", "put", "get", "put", "get", "put", "get", "get", "get"]
[[2], [1, 1], [2, 2], [1], [3, 3], [2], [4, 4], [1], [3], [4]]
```

**输出:**

```
[null, null, null, 1, null, -1, null, -1, 3, 4]
```

**解释:**

- `LRUCache lRUCache = new LRUCache(2);` // 容量为 2
- `lRUCache.put(1, 1);` // 缓存是 {1=1}
- `lRUCache.put(2, 2);` // 缓存是 {1=1, 2=2}
- `lRUCache.get(1);` // 返回 1，{1=1} 变为最近使用，缓存是 {2=2, 1=1}
- `lRUCache.put(3, 3);` // 容量已满，逐出最近最少使用的 {2=2}，缓存是 {1=1, 3=3}
- `lRUCache.get(2);` // 返回 -1 (因为 2 已被逐出)
- `lRUCache.put(4, 4);` // 容量已满，逐出 {1=1}，缓存是 {3=3, 4=4}
- `lRUCache.get(1);` // 返回 -1
- `lRUCache.get(3);` // 返回 3，缓存是 {4=4, 3=3}
- `lRUCache.get(4);` // 返回 4，缓存是 {3=3, 4=4}

## 设计思路：哈希表 + 双向链表

这个问题的难点在于 `get` 和 `put` 操作都必须是 O(1) 时间复杂度。

- **快速查找**: `get` 操作要求我们能快速地通过 `key` 找到 `value`。这立刻让我们想到了**哈希表 (HashMap / Map)**，它提供了 O(1) 时间复杂度的查找。
- **有序存储**: 我们还需要一个数据结构来维护所有缓存项的“最近使用”顺序。
  - 当一个项被 `get` 或 `put` (更新) 时，它需要被移动到“队尾”，表示“最近使用”。
  - 当缓存满了需要逐出时，我们需要从“队头”移除“最近最少使用”的项。
  - 这意味着我们需要一个支持在**头部 O(1) 删除**和**尾部 O(1) 插入/移动**的数据结构。**双向链表**是完美的选择。

**最终的数据结构组合：**

1.  **`Map<number, DLinkedNode>` (哈希表)**:
    - `key` 是缓存的键。
    - `value` 是一个指向**双向链表中对应节点的引用**。
    - 作用：实现 O(1) 的 `get` 查找。

2.  **`DLinkedNode` (双向链表节点)**:
    - 每个节点包含 `key`, `value`, `prev` (前驱指针), `next` (后继指针)。
    - `key` 是冗余存储，但在删除节点时非常有用（可以通过节点快速找到 Map 中的 key）。
    - `prev` 和 `next` 指针构成了链表。

3.  **双向链表本身**:
    - 我们使用两个“哨兵”节点 `head` 和 `tail` 来简化边界处理。`head` 指向最旧的节点，`tail` 指向最新的节点。
    - `head.next` 是真正的第一个节点（最近最少使用）。
    - `tail.prev` 是真正的最后一个节点（最近使用）。
    - 链表负责维护 LRU 顺序。

### `get(key)` 操作

1. 通过 Map 查找 `key`。如果不存在，返回 `-1`。
2. 如果存在，获取对应的 `node`。
3. **关键**: 将 `node` 从其当前位置移动到链表的尾部（表示最近使用）。
4. 返回 `node.value`。

### `put(key, value)` 操作

1. 通过 Map 查找 `key`。
2. **如果 `key` 存在**:
   - 获取 `node`，更新 `node.value`。
   - 将 `node` 移动到链表的尾部。
3. **如果 `key` 不存在**:
   - 创建一个新的 `DLinkedNode`。
   - 将新节点添加到链表的尾部。
   - 将 `[key, newNode]` 存入 Map。
   - **检查容量**: 如果 Map 的大小超过 `capacity`，则：
     - 获取链表头部的第一个真实节点 `head.next` (即最旧的节点)。
     - 从链表中删除这个节点。
     - 从 Map 中删除这个节点的 `key`。
