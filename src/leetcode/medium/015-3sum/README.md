# 15. 3Sum (三数之和)

## 题目描述

给你一个包含 `n` 个整数的数组 `nums`，判断 `nums` 中是否存在三个元素 `a`，`b`，`c` ，使得 `a + b + c = 0` ？请你找出所有和为 0 且不重复的三元组。

**注意：** 答案中不可以包含重复的三元组。

## 示例

**示例 1：**

- **输入:** `nums = [-1,0,1,2,-1,-4]`
- **输出:** `[[-1,-1,2],[-1,0,1]]`

**示例 2：**

- **输入:** `nums = []`
- **输出:** `[]`

**示例 3：**

- **输入:** `nums = [0]`
- **输出:** `[]`

## 约束条件

- `0 <= nums.length <= 3000`
- `-10^5 <= nums[i] <= 10^5`

---

## 解题思路：排序 + 双指针

这道题最容易想到的暴力解法是三层循环，时间复杂度为 O(n³)，在 n=3000 的情况下一定会超时。我们需要更优化的方法。

这道题可以看作是 "两数之和" 的升级版。我们可以通过固定一个数，将问题转化为在剩余数组中寻找 "两数之和" 的问题。

### 核心思想

1.  **排序**: 首先，对整个数组进行升序排序。排序是使用双指针法的前提，它有两个巨大的好处：
    - 方便我们移动指针来调整和的大小。
    - 方便我们进行去重操作。

2.  **遍历与固定**: 遍历排序后的数组，用一个 `for` 循环来固定第一个数 `nums[i]`。

3.  **双指针**: 对于每个固定的 `nums[i]`，我们在它后面的子数组 `[i+1, n-1]` 中使用**双指针**来寻找另外两个数。
    - 初始化左指针 `left = i + 1` 和右指针 `right = n - 1`。
    - 计算三个数的和 `sum = nums[i] + nums[left] + nums[right]`。
    - 根据 `sum` 与 `0` 的大小关系来移动指针：
      - 如果 `sum < 0`，说明和太小了，需要增大。因为数组是有序的，所以我们移动左指针 `left++`。
      - 如果 `sum > 0`，说明和太大了，需要减小。移动右指针 `right--`。
      - 如果 `sum === 0`，我们找到了一个符合条件的三元组！将 `[nums[i], nums[left], nums[right]]` 加入结果集。然后，为了找到其他可能的解，需要同时移动 `left` 和 `right` 指针。

### 关键：如何去重？

去重是这道题的难点和核心。我们需要处理两种重复情况：

1.  **第一个固定的数 `nums[i]` 重复**:
    - 当 `i > 0` 时，如果 `nums[i] === nums[i-1]`，说明这个数我们已经作为固定数处理过了，所有以它开头的解都已经被找到。为了避免重复，我们应该直接 `continue`，跳过本次循环。

2.  **双指针找到的解 `nums[left]` 和 `nums[right]` 重复**:
    - 当 `sum === 0` 并且我们找到了一个解之后，`left` 和 `right` 指针不能简单地 `++` 和 `--`。
    - 考虑 `[-2, 0, 0, 2, 2]`，固定 `-2` 后，`left` 指向第一个 `0`，`right` 指向最后一个 `2`，`sum=0`，找到 `[-2, 0, 2]`。
    - 此时，我们需要持续移动 `left` 指针，跳过所有与 `nums[left]` 相等的后续元素 (`while (left < right && nums[left] === nums[left+1]) left++;`)。
    - 同样，我们也需要持续移动 `right` 指针，跳过所有与 `nums[right]` 相等的后续元素。
    - 完成这些去重操作后，再将 `left++` 和 `right--`，继续寻找新的可能性。

### 优化

- 如果在遍历 `nums[i]` 时，发现 `nums[i] > 0`，由于数组已经排序，后面的 `nums[left]` 和 `nums[right]` 也都大于等于 `nums[i]`，它们的和不可能等于 `0`。因此，可以直接 `break` 循环，提前结束。

## 复杂度分析

- **时间复杂度**: O(n²)。排序 O(n log n)，外层循环 O(n)，内层双指针 O(n)，所以总体是 O(n²)。
- **空间复杂度**: O(log n) 或 O(n)，取决于排序算法内部实现所需的空间。
