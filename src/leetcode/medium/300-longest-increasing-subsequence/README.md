# 300. Longest Increasing Subsequence (最长递增子序列)

## 题目描述

给你一个整数数组 `nums` ，找到其中最长严格递增子序列的长度。

**子序列** 是由数组派生而来的序列，删除（或不删除）数组中的元素而不改变其余元素的顺序。例如，`[3,6,2,7]` 是数组 `[0,3,1,6,2,2,7]` 的子序列。

## 示例

**示例 1：**

- **输入:** `nums = [10,9,2,5,3,7,101,18]`
- **输出:** `4`
- **解释:** 最长递增子序列是 `[2,3,7,101]`，因此长度为 4。

**示例 2：**

- **输入:** `nums = [0,1,0,3,2,3]`
- **输出:** `4`
- **解释:** 最长递增子序列是 `[0,1,2,3]`。

**示例 3：**

- **输入:** `nums = [7,7,7,7,7,7,7]`
- **输出:** `1`
- **解释:** 严格递增，所以最长的是单个元素。

## 约束条件

- `1 <= nums.length <= 2500`
- `-10^4 <= nums[i] <= 10^4`

---

## 解法一：动态规划 (Dynamic Programming)

这是比较符合直觉的解法。

### 核心思想

我们定义一个 `dp` 数组，`dp[i]` 表示**以 `nums[i]` 这个数结尾**的最长递增子序列的长度。

为了计算 `dp[i]`，我们需要回头看 `i` 之前的所有元素 `nums[j]` (其中 `0 <= j < i`)。

- 如果 `nums[i] > nums[j]`，这意味着 `nums[i]` 可以接在以 `nums[j]` 结尾的递增子序列的后面，形成一个更长的递增子序列。
- 此时，以 `nums[i]` 结尾的递增子序列的长度，就可能是 `dp[j] + 1`。

由于 `j` 可以取 `0` 到 `i-1` 的任何值，我们需要遍历所有可能的 `j`，并找到最大的那个 `dp[j]`。所以，状态转移方程是：
`dp[i] = max(dp[j]) + 1`，其中 `0 <= j < i` 且 `nums[j] < nums[i]`。

**基础情况**: 每个元素自身都可以构成一个长度为 1 的递增子序列，所以 `dp` 数组的所有元素初始值都为 `1`。

**最终结果**: `dp` 数组中的最大值就是整个数组的最长递增子序列的长度（因为最长子序列不一定以最后一个元素结尾）。

### 算法步骤

1. 创建一个长度与 `nums` 相同的 `dp` 数组，并用 `1` 填充。
2. 从第二个元素开始遍历 `nums` (索引 `i` 从 1 到 `n-1`)。
3. 对于每个 `i`，再嵌套一个循环，遍历它之前的所有元素 (索引 `j` 从 0 到 `i-1`)。
4. 在内层循环中，如果 `nums[i] > nums[j]`，则更新 `dp[i] = Math.max(dp[i], dp[j] + 1)`。
5. 所有循环结束后，返回 `dp` 数组中的最大值。

### 复杂度分析

- **时间复杂度**: O(n²)。因为有两层嵌套循环。
- **空间复杂度**: O(n)。需要一个 `dp` 数组。

## 解法二：贪心 + 二分查找 (Patience Sorting)

这是一种更巧妙且高效的解法，思路源于一种叫做“耐心排序”的游戏。

### 核心思想

我们维护一个数组 `tails`，`tails[i]` 存储的是所有长度为 `i+1` 的递增子序列中，**结尾元素最小的那个值**。

这个 `tails` 数组有一个非常重要的性质：**它一定是严格递增的**。

> _（反证法：如果 `tails[i] >= tails[i+1]`，那么长度为 `i+2` 的子序列的结尾元素 `tails[i+1]` 比长度为 `i+1` 的子序列的结尾元素 `tails[i]` 还要小。我们可以用这个更小的 `tails[i+1]` 去替换 `tails[i]`，从而得到一个结尾元素更小的、长度为 `i+1` 的子序列。这与 `tails[i]` 的定义矛盾。）_

现在，我们遍历 `nums` 数组中的每一个数字 `num`：

1. **如果 `num` 大于 `tails` 数组中所有的元素** (即 `num > tails[tails.length - 1]`)：
   - 这意味着我们可以把 `num` 接在最长的递增子序列后面，形成一个更长的子序列。
   - 我们就直接把 `num` 添加到 `tails` 数组的末尾。`tails` 数组的长度加一。

2. **如果 `num` 小于或等于 `tails` 数组中的某个元素**：
   - 我们需要在 `tails` 数组中，找到**第一个大于或等于 `num` 的元素 `tails[i]`**，然后用 `num` **替换**掉它。
   - 这个操作的意义是：我们发现了一个新的、长度为 `i+1` 的递增子序列，它的结尾元素 `num` 比我们之前记录的 `tails[i]` 还要小。根据贪心思想，**更小的结尾元素意味着未来有更大的潜力接纳更多的数**。所以我们更新它。

**最终结果**: `tails` 数组的**长度**就是最长递增子序列的长度。`tails` 数组本身并不是那个子序列！

由于 `tails` 数组是单调递增的，上述“查找第一个大于或等于 `num` 的元素”的操作，可以用**二分查找**来高效完成。

### 复杂度分析

- **时间复杂度**: O(n log n)。遍历 `nums` 是 O(n)，每次遍历中的二分查找是 O(log n)。
- **空间复杂度**: O(n)。需要一个 `tails` 数组。
