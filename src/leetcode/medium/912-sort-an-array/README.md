# 912. Sort an Array (排序数组)

## 题目描述

给你一个整数数组 `nums`，请你将该数组升序排列。

## 示例

**示例 1：**

- **输入:** `nums = [5,2,3,1]`
- **输出:** `[1,2,3,5]`

**示例 2：**

- **输入:** `nums = [5,1,1,2,0,0]`
- **输出:** `[0,0,1,1,2,5]`

## 约束条件

- `1 <= nums.length <= 5 * 10^4`
- `-5 * 10^4 <= nums[i] <= 5 * 10^4`

---

## 解法一：快速排序 (Quick Sort)

快速排序是一种非常高效的、不稳定的、原地排序算法。

### 核心思想 (分治)

1.  **分解 (Divide)**: 从数组中选择一个元素作为“基准”(pivot)。将数组重新排列，使得所有小于基准的元素都放在基准前面，所有大于基准的元素都放在基准后面。这个过程称为“分区”(partition)。分区结束后，基准元素就位于其最终排序后的正确位置。
2.  **解决 (Conquer)**: 递归地对基准左右两边的子数组进行快速排序。
3.  **合并 (Combine)**: 因为是原地排序，当递归的子数组都有序后，整个数组自然就有序了，无需额外的合并步骤。

### 分区 (Partition) 的实现

最经典的分区实现是 "Lomuto partition scheme"：

1. 选择数组最后一个元素作为基准 `pivot`。
2. 维护一个指针 `i`，它指向最后一个小于 `pivot` 的元素的位置，初始为 `left - 1`。
3. 用另一个指针 `j` 遍历从 `left` 到 `right - 1` 的数组部分。
4. 如果 `nums[j] < pivot`，则 `i++`，然后交换 `nums[i]` 和 `nums[j]`。这确保了所有小于 `pivot` 的元素都被移动到数组的左侧。
5. 遍历结束后，交换 `nums[i + 1]` 和 `pivot` (即 `nums[right]`)。这样，`pivot` 就被放到了它最终的正确位置 `i + 1`。
6. 返回 `pivot` 的最终索引 `i + 1`。

### 优化

- **随机化基准**: 为了避免在数组近乎有序时导致 O(n²) 的最坏情况，每次分区前，随机选择一个元素与最后一个元素交换，再将最后一个元素作为基准。

### 复杂度分析

- **平均时间复杂度**: O(n log n)
- **最坏时间复杂度**: O(n²) (当基准选择最差，每次都划分为 n-1 和 0 的子数组时)
- **空间复杂度**: O(log n) (递归调用栈的深度)

## 解法二：归并排序 (Merge Sort)

归并排序是一种非常高效的、稳定的、非原地排序算法。

### 核心思想 (分治)

1.  **分解 (Divide)**: 将长度为 n 的数组不断对半切分，直到每个子数组只有一个元素（长度为1的数组天然有序）。
2.  **解决 (Conquer)**: 这一步在归并排序中实际上是空的，因为分解的最终结果已经是“解决”了的（单个元素有序）。
3.  **合并 (Combine)**: 递归地将两个已排序的子数组**合并 (merge)** 成一个更大的有序数组。这是归并排序的核心操作。

### 合并 (Merge) 的实现

1. 创建一个临时数组 `temp` 来存放合并结果。
2. 使用两个指针 `i` 和 `j`，分别指向两个待合并的子数组 `leftArr` 和 `rightArr` 的起始位置。
3. 比较 `leftArr[i]` 和 `rightArr[j]` 的大小，将较小的那个元素放入 `temp`，并将对应的指针后移。
4. 重复步骤 3，直到其中一个子数组遍历完毕。
5. 将另一个子数组中剩余的所有元素直接追加到 `temp` 的末尾。
6. 将 `temp` 中的元素复制回原数组的相应位置。

### 复杂度分析

- **时间复杂度**: O(n log n) (无论是最好、最坏还是平均情况，都非常稳定)
- **空间复杂度**: O(n) (需要一个与原数组等大的临时数组用于合并)
